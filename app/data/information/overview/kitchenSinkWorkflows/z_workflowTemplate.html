If the workflow for adding/updating/importing instructs you to implement a new component for the UI-Kitchen-Sink you MUST provide a:

JSON-File
Use the template bellow to create the JSON-File named {idOfComponent}.json describing your new component (see template and description bellow).
HTML-File
DOM structure of the new component, named {idOfComponent}.html

If custom less or JS code is needed to render the component you MUST also provide:

LESS-File
Containing the needed less classes, named {idOfComponent}.html
JS-File
Containing the JS-Logic, named {idOfComponent}.html

Description of JSON (see bellow for the template):

{
        "id": "id of the new component, entries will be sorted according the component id. All components must have as first class in the dom the name of the il-{id-of-component-separeted-by-hyphens} (REQUIRED).",
        "title": "Name of the component, appearing in the tab section (REQUIRED). The name should match the id but may reverse the order of words if making the subject more readable.",
        "description": "Some useful description. The context of where and how to use this component should be made clear. (REQUIRED)",
        "state": "State of the component (see 'States' for more information about states) (REQUIRED)",
        "lessVariables (OPTIONAL)":
        [
            {
                "title": "title of the less variable used in this component. Due only list variables here, that are not already listed in subelements that MUST or MAY be used by some sub component.",
                "description": "Description of how this variable is used in this component.",
                "state": "State of the usage of this variable in this component"
            }
        ],
        "guiClass": "GUI-Class in ILIAS that is used to render this component (OPTIONAL)",
        "relations (OPTIONAL)": {
                "isA": "id of Parent. Each Element can only have one",
                "mustUse":[
                    "id of components that MUST be placed inside the described component"
                ],
                "mayUse":[
                    "id of components that MAY be placed inside the described component"
                ]
        }
}
