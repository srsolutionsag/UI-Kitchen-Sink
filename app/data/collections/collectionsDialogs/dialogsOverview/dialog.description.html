In the WAI-ARIA roles model 'dialogs' are described as 'an application window that is designed to interrupt the current processing of an application in order to prompt the user to enter information or require a response'.

Mozilla Dev-Guide further explains that dialogs can either be 'non-modal (it's still possible to interact with content outside of the dialog) or modal (only the content in the dialog can be interacted with)'. Marking up a dialog element with the dialog role helps assistive technology identify the dialog's content as being grouped and separated from the rest of the page content. However, adding role="dialog" alone is not sufficient to make a dialog accessible. Additionally, the following needs to be done:

  • The dialog must be properly labeled
  • Keyboard focus must be managed correctly
