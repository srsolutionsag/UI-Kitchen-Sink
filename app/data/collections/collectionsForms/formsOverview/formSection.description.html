A section is composed of an identifier on the left and a form element on the right. A description of the setting configured in a section MIGHT be added bellow the form element on the right side.

Consider the following key aspects for the wording of the identifier (Bezeichner):

  • An identifier SHOULD be composed of one single term or a short expression. Elaborate description belong into the description. They identifier works as an eye catcher by browsing over it.
  • You should use short words instead of long construct in the identifier.
  • You MUST be consistent with terms that exist in other objects. E.g. "availability" MUST NOT be renamed to "availableness". If you feel a wording needs to be changed, then you MUST propose it to the JF.
  • Good Example: Authentication
  • Bad Example: Authentication method used by a login of a user

Consider the following key aspects for the wording of the description (Beschreibung):

  • Description are OPTIONAL. You SHOULD make sure, that a description is really needed. A description that just repeats the wording of the identifier or wording used in the form element is unnecessary. E.g. The sections "Title" and "Description" in the "Settings" tab of modules to not need a description.
  • In a section with many options (such as radio buttons) the description MUST be placed directly underneath to option it describes.
  • You MUST NOT address the user directly in your description text.
  • You MUST express full sentences with a period (.) at the end.
  • You SHOULD phrase short sentences with no more than 25 words.
  • You SHOULD NOT use any formatting in descriptions (bold, italic or similar).
  • You SHOULD NOT parenthesize expressions in your descriptions.
  • You SHOULD NOT only express what a certain setting effects but also explain, why this might be important and what it can be used for.
  • You SHOULD NOT start your description with terms such as:
    • If this option is set ...
    • If this setting is active ...
    • Chose this setting if ...
    • This setting ...
  • Good Example: A course set to online is only available in the defined time range.
  • Bad Example: Chose this setting to set to course online for the defined time range only.
